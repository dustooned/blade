🧩 Blade_TH_v1 – Project Overview & Learnings

📦 Blade Layout System

We used a modular .blade container system with a flex-based layout inside .blade-content. This allowed us to build each page with a consistent structure using two groups: .text-group and .visual-group.

🔄 Procedural Alternating Layout

To alternate layouts automatically, we used the :nth-child(even) selector in CSS. This flips the flex direction of every second blade using:

.blade:nth-child(even) .blade-content {
  flex-direction: row-reverse;
}
🔠 Reveal Text Animation

Each block of text appears using a vertical mask reveal effect. This was built using IntersectionObserver and a CSS @keyframes animation applied to .text-reveal when its parent blade enters the viewport.

🧭 Navigation Arrows

Top and bottom arrows were added using Unicode characters ↑ and ↓. They were made hover-animated with CSS bounce effects using @keyframes and positioned with fixed CSS layout.

📐 Responsive & Expanding Blades

To allow long content or stacked elements to expand naturally, we switched from height: 100vh to min-height: 100vh on each .blade. This makes each blade start full screen but grow vertically if needed.

📚 What We Learned
  • 💡 How to structure HTML containers for flexible layouts
  • 🎨 How to animate elements with scroll-triggered observers
  • 🧱 How to build modular, reusable page sections
  • ⚡ How to avoid clipping issues using min-height
  • 🎛️ How to apply automatic alternating layouts with no JS
